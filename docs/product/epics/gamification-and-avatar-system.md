# Epic - Gamification and Avatar System

## Goal

Increase student engagement through consistent, healthy and learning-aligned gamification.

Gamification should reward effort, consistency, improvement and mastery without creating unhealthy pressure.

## Principles

1. Reward progress, not only high performance.
2. Reward consistency.
3. Avoid public shame.
4. Avoid excessive competition for younger students.
5. Make achievements educationally meaningful.
6. Keep visual identity fun and safe.
7. Avoid copyrighted characters.
8. Support institutional enablement/disablement.

## Points Model

Boom should separate points into at least two categories.

### Engagement Points

Reward participation and habit.

Examples:

- completing the daily activity;
- maintaining a learning streak;
- reviewing a past mistake;
- returning after a missed day;
- finishing within the expected time window;
- completing a reflection question.

### Performance Points

Reward mastery and improvement.

Examples:

- answering correctly;
- improving compared to previous attempts;
- mastering a skill;
- closing a gap;
- solving a harder question;
- maintaining high confidence with accuracy.

### Growth Points

Optional future category.

Reward improvement even when absolute score is still low.

Examples:

- moving from 40% to 60% accuracy;
- reducing response time without losing accuracy;
- improving confidence calibration;
- reducing repeated misconceptions.

## Avatar System

Boom should support original avatar themes instead of known copyrighted heroes.

Recommended avatar archetypes:

- Explorer;
- Guardian;
- Mage;
- Engineer;
- Scientist;
- Time Traveler;
- Space Pilot;
- Forest Ranger;
- Cyber Hero;
- Inventor;
- Scholar;
- Navigator;
- Builder;
- Alchemist.

## RPG Theme

The platform may represent learning as a journey.

Examples:

```text
Learning Journey
  -> World
    -> Region
      -> Quest
        -> Challenge
          -> Skill
```

A mathematics journey could be represented as a map. Each topic unlocks new areas after enough evidence of mastery.

## Rewards

Possible rewards:

- avatar items;
- visual badges;
- titles;
- profile frames;
- new map areas;
- quest completion;
- streak badges;
- subject mastery badges.

## Leaderboards

Leaderboards should be optional and admin-controlled.

Possible modes:

- disabled;
- private only;
- classroom anonymous ranking;
- team-based;
- improvement-based;
- no ranking, only personal progress.

## MVP

The first version should support:

1. Points ledger.
2. Daily streak.
3. Basic achievements.
4. Avatar profile.
5. Reward rules.
6. Admin setting to enable/disable gamification.
7. No public leaderboard initially.
