# Epic - Parent / Guardian Dashboard

## Goal

Provide parents/guardians with a simple, attractive and useful dashboard showing learning activity history, performance and the current action plan.

## Business Rationale

Parents do not need complex administrative tools. They need clarity.

The parent dashboard should answer:

- Did the student study?
- How often?
- For how long?
- How well?
- In which subjects?
- What is improving?
- What needs attention?
- What is the current plan?

## Product Scope

The dashboard should include:

1. Student selector.
2. Weekly activity summary.
3. Activity history chart.
4. Performance overview.
5. Subject performance.
6. Learning gaps.
7. Current action plan.
8. Recent activity timeline.

## UX Principle

```text
Simple surface.
Deep data underneath.
```

The user interface should not expose unnecessary complexity. It should show summarized insights, while the backend preserves detailed evidence for analytics and AI.

## MVP Dashboard Sections

### Weekly Summary

- Completed activities.
- Total study time.
- Average accuracy.
- Current streak.

### Main Chart

- Daily activity completion.
- Accuracy trend.
- Study time trend.

### Performance by Subject

- Subject name.
- Accuracy.
- Study time.
- Trend indicator.

### Attention Areas

- Topic/skill name.
- Severity.
- Days open.
- Practice time.
- Current status.

### Current Action Plan

- Plan title.
- Target topic/skill.
- Recommended next steps.
- Estimated effort.
- Progress status.

## Non-Goals

The MVP dashboard does not include:

- Complex admin configuration.
- Chat interface.
- Full AI assistant.
- Social ranking.
- Gamification map.
- Mobile app.

## Success Metrics

- Parent understands the student's recent activity in less than one minute.
- Parent can identify the current attention area.
- Parent can see the current action plan.
- Parent does not need to interpret raw answer-level data.
